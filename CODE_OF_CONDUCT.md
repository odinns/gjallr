# Code Of Conduct

Be direct. Be useful. Do not be an ass.

## Expected

- Keep critique about the work.
- Explain risk clearly.
- Respect project boundaries.
- Prefer concrete examples over vague disapproval.

## Not Accepted

- Harassment or personal attacks.
- Bigotry.
- Publishing private security details.
- Turning technical disagreement into theatre.

Maintainers may remove issues, comments, or contributors that make the project worse to work on.
